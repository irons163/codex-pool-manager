# CodexPoolManager v1.0.11

リリース日：2026-06-08

## 主な変更点

- API key を使う中継 provider へ Codex CLI を手動で切り替えられる API Key 中継アカウントを追加しました。
- 認証フローを OAuth / サブスクリプションアカウントと API Key 中継アカウントの 2 つに分け、分かりやすくしました。
- Codex の既存履歴を表示したまま API リクエストを中継 Base URL に送る、履歴保持モードを追加しました。
- 中継アカウントからサブスクリプションアカウントへ戻す際に、OAuth 認証情報と provider 設定が正しく復元されるよう修正しました。
- API key 中継アカウントはアカウント一覧の最後に並び、使用量同期と自動切り替えの対象外になります。
- 中継設定フォームを改善しました。Base URL は主要な必須項目になり、API形式には説明を追加し、Base URL は初期値を空にしました。
- 中継アカウント UI とリリースノートの多言語対応を追加しました。

## 注意事項

- API key 中継アカウントは ChatGPT サブスクリプションの使用量データを提供しないため、手動切り替えのみ対応します。
- 履歴保持モードは、次に中継アカウントへ切り替える時に適用されます。
- 手動の移行作業は不要です。
