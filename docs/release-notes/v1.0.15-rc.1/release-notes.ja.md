# CodexPoolManager v1.0.15-rc.1

リリース日：2026-07-11

## 改善

- リセットクレジットの有効期限は、前回の同期時刻から推定せず、Codex アカウント詳細エンドポイントが返す `expires_at` を直接使用するようになりました。
- アカウントカードとリセット通知には、取得できる場合は API の正確な日付が表示されます。
- API から取得した日付には推定日の警告を表示せず、フォールバック推定を使用する場合にのみ警告を表示します。
- 有効期限の取得元をアカウントスナップショットに保存し、再起動後も API の正確な日付と推定日を区別します。
- README と対応するすべての UI 言語を更新し、API の正確な日付とフォールバック推定を一貫して説明します。

## 互換性

- アカウント詳細または有効期限を取得できない場合は、既存の推定処理をフォールバックとして使用します。
- 旧バージョンのスナップショットは引き続き読み込め、安全のため推定データとして扱われます。

## Prerelease の注記

- この prerelease では、次回の stable リリース前に正確な有効期限の同期とデータ取得失敗時のフォールバックを検証します。
